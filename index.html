AI 활용 개인 맞춤 여행 초안 생성 서비스

Web

설명: AI가 사용자 선호도를 분석해 여행 초안을 추천하는 서비스입니다.
인원: 3인 개발
언어/환경: React, NodeJS, Express
개발 기간: 2025.09.03 ~ 2025.11.12

Problem (Challenge)

  1. LLM이 “영국에서 유명한 스시를 먹어보세요.”와 같이 지역 맥락과 맞지 않는 정보를 생성하는 할루시네이션 문제가 발생했습니다.
  2. 데이터 처리 구조 측면에서도 문제를 겪었습니다. 여행 추천에 사용되는 사진 데이터의 메타정보(국가/카테고리/태그/설명 등)를 어떤 구조로 저장·관리할지가 명확하지 않았습니다.

How (Resolution)

  1. 프롬프트 구조 재설계와 모델 업그레이드를 통해 해결했습니다.
  2. 사진 데이터 관리를 위해 제목-내용.확장자 형태의 자체 메타데이터 규칙을 정의해 일관된 구조로 저장·처리했습니다. 그 결과 데이터 분류와 조회 과정이 단순화되었고, 실제 개발 일정에서도 약 5일 정도 기간을 단축할 수 있었습니다.

Retrospective

[개인 의견]
3인 이상의 팀으로 개발한 경험이 처음이었기에, 초기에는 역할 분담과 개발 규칙을 정립하는 과정이 다소 낯설게 느껴졌습니다. 하지만 공통 규칙과 개발 흐름을 정립한 이후에는 개인 개발보다 높은 효율과 시간 절약 효과를 체감할 수 있었고, 팀원 간의 상호 자극을 통해 개발 동기 또한 크게 향상되었습니다. 이 경험을 통해 팀 기반 개발의 가치를 분명히 느끼게 되었으며, 이후에는 개인 개발보다는 팀 프로젝트 중심으로 경험을 쌓고자 하는 목표를 가지게 되었습니다.

[아쉬웠던 점]
다만 아쉬운 점은 사용자 관점의 UX 디자인이 충분히 고려되지 못했다는 부분입니다. 개발자 3명으로 구성된 팀 특성상 시스템 흐름과 구현 중심의 설계에 치중한 결과, 전시회 당시 교수님으로부터 “사용자 입장에서의 직관성이 부족하다”는 피드백을 받았습니다. 이 피드백은 프로젝트 종료 이후까지도 깊이 남았으며, 이후 개발에서는 기능 구현뿐 아니라 UX 관점에서의 설계까지 함께 고려하는 개발자로 성장하고자 하는 계기가 되었습니다.

고양이 키우기

Unity 2D · Mobile Idle RPG

설명: 방치형 구조를 기반으로 고양이를 합성해 높은 등급의 고양이를 획득하는 모바일 게임
인원: 1인 개발
언어/환경: C#, Unity 2D
개발 기간: 2025.08.03 ~ 2025.10.08

Problem (Challenge)

  1. 중반에서 후반부로 넘어갈 때 콘텐츠 소모 속도를 조절하면 정체 구간이 발생해 유저 이탈 문제가 발생했습니다.
  2. 화면 크기에 따라 UI 깨짐 문제가 발생했으며, 특히 Pad 형식의 기기에서 문제가 두드러졌습니다.
  3. 고양이 리소스 부족으로 인해 캐릭터 다양성 확보에 한계가 발생하였습니다.

How (Resolution)

  1. 단순 수치 성장 대신 합성 확률, 대박 시스템, 유물 강화를 도입해 후반까지 이어질 수 있는 성장 사다리를 설계했습니다.
  2. 기기의 물리적 화면 크기(DPI)를 기준으로 1차 판별하고, 보조적으로 화면비를 적용해 안정성을 높였습니다.
  3. 해당 문제는 고양이 리소스를 직접 제작하는 방식으로 해결했습니다. 기존 고양이 캐릭터를 참고하되 그대로 사용하지 않고, 이왕 그리는거 세상에 없는 고양이로 그려보았습니다. EX) 마왕 고양이, 요리사 고양이, 우주 비행사 고양이

Retrospective

[개인 의견]
이전 작품인 나만의 작은 용사를 개발하며 느꼈던 문제점과 아쉬운 부분을 보완하고자 고양이 키우기 게임을 개발하게 되었습니다. 이번 프로젝트에서는 기획자나 다른 개발자가 제 작업을 보았을 때 커뮤니케이션이 원활하게 이루어질 수 있도록, Header, Tooltip, Min/Max 값 등 코드와 기획 정보를 명확히 정의하는 데 중점을 두고 개발했습니다. 초기에는 작업량이 늘어나 번거롭게 느껴졌지만, 프로젝트가 진행될수록 구조 파악과 수정이 훨씬 수월해졌고, 결과적으로 유지보수 측면에서 큰 장점으로 작용했습니다.

[아쉬웠던 점]
고양이 리소스 수량을 사전에 충분히 파악하지 못했던 점이 아쉬움으로 남았습니다. 9월달에 제작 완료를 목표로 데드라인을 설정했으나, 리소스 부족으로 인해 추가 제작이 필요해 결과적으로 약 1주일 정도 개발 일정이 연장되었습니다.

나만의 작은 용사

Unity 2D · Mobile Idle RPG

설명: 방치형 구조로 용사를 키워 최종 보스인 마왕을 무찌르면 되는 게임
인원: 1인 개발
언어/환경: C#, Unity 2D
개발 기간: 2024.02.01 ~ 2024.06.16

Problem (Challenge)

  1. 각 난이도별 클리어 시 난이도 전환 및 스테이지 이동 시 표시 오류가 발생하는 문제가 있었습니다.

How (Resolution)

  1. 저는 상태변수 기반 구조로 문제를 해결했습니다.맵 구성 자체가 2차원 구조이기 때문에 이를 활용해 PinInStage[][]와 MoveInStage[][]라는 두 개의 2차원 배열 상태변수를 설계했습니다. 각 배열은 난이도–스테이지 조합의 상태를 정확하게 추적하며, 스테이지 이동·조회·난이도 전환 시 해당 배열의 상태값만 갱신하도록 만들었습니다.

Retrospective

[개인 의견]
첫 방치형 RPG 게임을 개발하며 다양한 시행착오를 경험했습니다. 다수의 Prefab을 활용하는 과정에서, 데이터 구조를 사전에 얼마나 촘촘하게 설계하느냐에 따라 개발 중 발생하는 불편함이 크게 달라진다는 점을 체감했습니다. 또한 단순한 조작을 중심으로 한 Jump Action 게임보다 방치형 RPG가 시스템과 데이터 측면에서 훨씬 많은 작업량을 요구한다는 점도 알게 되었습니다.

[아쉬웠던 점]
이번 프로젝트를 통해 게임 제작 자체에 대한 즐거움을 분명히 느낄 수 있었지만, 동시에 ‘잘 만든 게임’이라고 말하기에는 아직 부족하다는 한계도 인식하게 되었습니다. 이러한 경험을 바탕으로 다음 프로젝트에서는 보다 세밀한 기획과 구조 설계를 바탕으로 개발할 수 있는 기획과 구현을 함께 고려하는 개발자로 성장하고자 합니다.
